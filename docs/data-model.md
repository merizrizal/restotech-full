No local migrations or model classes are present in this repository. The following model is inferred from controller imports, query relations, view fields, and table names referenced by SQL/query builder code.

## Core table/room entities

### `MtableCategory` / table `mtable_category`

Purpose: room/table category (`Ruangan`).

Observed fields: `id`, `nama_category`, `color`, `image`, `keterangan`, `not_active`, `is_deleted`, audit fields.

Relations referenced:

- `mtables`

Used by room layout, table browsing, and backend room CRUD.

### `Mtable` / table `mtable`

Purpose: physical table.

Observed fields: `id`, `mtable_category_id`, `nama_meja`, `kapasitas`, `status`, `keterangan`, `not_active`, `not_ppn`, `not_service_charge`, `image`, `shape`, `layout_x`, `layout_y`, `is_deleted`, audit fields.

Relations referenced:

- `mtableCategory`
- `mtableSessions`

Used by frontend POS, table layout editor, booking/opening sessions.

### `MtableSession`

Purpose: active/closed customer seating session.

Observed fields from controllers: `id`, `mtable_id`, `nama_tamu`, `jumlah_tamu`, `jumlah_harga`, `discount`, `pajak`, `service_charge`, `is_closed`, `opened_at`, `user_opened`, `is_join_mtable`.

Relations referenced:

- `mtable`
- `mtableOrders`
- `mtableSessionJoin`
- `userOpened.kdKaryawan`

Used for open tables, split, transfer, join, sale invoice correction, and booking-open.

### `MtableJoin` and `MtableSessionJoin`

Purpose: group multiple table sessions into one active bill.

Observed fields:

- `MtableJoin::active_mtable_session_id`

Relations referenced:

- `MtableJoin.activeMtableSession`
- `MtableJoin.mtableSessionJoins`
- `MtableSessionJoin.mtableJoin`

### `MtableOrder`

Purpose: POS order line inside a table session.

Observed fields: `id`, `mtable_session_id`, `menu_id`, `catatan`, `harga_satuan`, `jumlah`, `discount_type`, `discount`.

Used by split/transfer/join flows and queue creation.

### `MtableOrderQueue`

Purpose: kitchen/order queue row.

Observed fields: `id`, `mtable_order_id`, `menu_id`, `jumlah`, `keterangan`, `is_finish`, `is_send`.

Relations referenced:

- `menu`
- `mtableOrder`
- `mtableOrder.mtableSession`

### `MtableBooking`

Purpose: booking/reservation.

## Stock/inventory entities

### `Item` and `ItemSku`

External item catalog models. Observed fields include `item_id`, `item_sku_id`, `nama_item`, `nama_sku`, `no_urut`, and SKU defaults for `storage_id`/`storage_rack_id` via menu recipes.

### `Storage` and `StorageRack`

Storage fields: `id`, `nama_storage`, `keterangan`.

Storage rack fields: `id`, `storage_id`, `nama_rak`, `keterangan`.

### `Stock`

Observed fields: `id`, `item_id`, `item_sku_id`, `storage_id`, `storage_rack_id`, `jumlah_stok`, audit fields.

Static methods used:

- `Stock::setStock($itemId, $skuId, $storageId, $rackId, $delta)`

Composite lookup in correction uses concatenated key: `$model->item_id . $model->item_sku_id . $model->storage_id . $model->storage_rack_id`.

### `StockMovement`

Observed fields: `id`, `tanggal`, `type`, `item_id`, `item_sku_id`, `storage_from`, `storage_rack_from`, `storage_to`, `storage_rack_to`, `jumlah`, `reference`, `keterangan`, audit fields.

Types observed:

- `Inflow`
- `Outflow`
- `Transfer`
- `Inflow-PO`
- `Outflow-RP`
- `Inflow-Convert`
- `Outflow-Convert`
- `Outflow-Menu`
- `Koreksi`

Static helpers used:

- `StockMovement::setInflow(...)`
- `StockMovement::setOutflow(...)`

### `StockKoreksi`

Observed fields: `id`, `date_action`, `user_action`, `item_id`, `item_sku_id`, `storage_id`, `storage_rack_id`, `jumlah_awal`, `jumlah`, `jumlah_adjustment`, `action`, audit fields.

Workflow: pending rows (`action = Waiting`) can be bulk-approved/rejected. Approval updates `Stock` and creates `StockMovement` type `Koreksi`.

## Finance/admin entities

Observed fields: `id`, `nama_payment`, `type`, `method`, `keterangan`, `not_active`, audit fields.

ID generated by `Settings::getTransNumber('id_payment_method', 'ym', $model->nama_payment)`.

### `TransactionAccount`

Observed fields: `id`, `nama_account`, `account_type`, audit fields.

`account_type` values observed: `Cash-In`, `Cash-Out`.

### `TransactionCash`

Observed fields: `id`, `account_id`, `date`, `jumlah`, `reference_id`, `keterangan`, audit fields.

### `Voucher`

Observed fields: `id`, `voucher_type`, `jumlah_voucher`, `start_date`, `end_date`, `not_active`, `keterangan`, audit fields.

Voucher types:

- `Percent` - discount value is percentage of bill.
- `Value` - fixed discount amount.

IDs generated by `Settings::getTransNumber('id_voucher', 'ym')`.

### `Employee`

Observed fields: primary key `kode_karyawan`; `sisa` employee limit.

Used for employee limit validation and payment correction.

### `Settings`

Static settings/sequence model. Methods observed:

- `Settings::getTransNumber(...)`
